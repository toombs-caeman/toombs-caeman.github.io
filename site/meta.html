Meta

You NEED to use Jekyll | Wordpress | Ghost | etc. in order to maintain your blog! -- Somebody, probably
Well if you want to enjoy the experience, then maybe. But if you'd rather watch on forlornly as sections of your site break for no apparent reason and spend hours upon hours debugging regular expressions when you could be watching grass grow then have I got just the thing for you.

Introducing blog.sh, a static site generator that's pure bs (bourne shell). As a single bash file with zero dependencies weighing in at 206 lines this is the best way to put your website rendering on a diet. I'm doing my part to fend off the Website Obesity Crisis.

This site is served using blog.sh and github pages. It is entirely static. There's no server side processing (pages are rendered during pre-commit), no client side javascript (not even for the burger animation, though blog.sh doesn't forbid that), and no tracking/analytics.

If I ever have to move away from github for some reason, the site could be hosted as simply as nginx or httpd serving an up-to-date copy of the repository. Nginx logs can be captured to provide unique visitors, which would be sufficient for me to see that people are interested without tracking your every move.

features

Templating

  • based on Markdown
    • blank lines converted to paragraph breaks
    • dashed lines converted to horizontal rules
    • headers
    • unordered lists (- or *)
    • links syntax works
  • preamble to set metadata (before ===)
  • macros
    • can insert bash script in between backticks (`). non-nestable
    • add helper functions in helpers.sh
  • extended tags

Special vars:

  • pages - an IFS seperated list of filenames to be rendered
  • page - refers to the current filename being rendered
  • url - refers to the current url being rendered
  • content - contains the contents when rendering a layout

Page scoped vars:

  • actually stored globally with a prefix like "page_page name_var_name"

References

  • omitting html tags
  • bash regex
  • bash parameter expansion
